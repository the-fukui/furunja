# ふるんじゃ

スマホを縦持ちのまま前後にかたむけて音程を操作する、テルミン風のWebオタマトーン。

## あそびかた

1. スマホでページを開き、「ならす」をタップする
2. センサーの利用を許可する（iOSのみダイアログが出る）
3. 本体を水平にすると真ん中の音（A4）、手前に起こすと高く、逆にかたむけると低くなる
4. 可動域は水平から±45度で、2オクターブ（A3〜A5）
5. 音は半音にスナップしつつ、間はポルタメントで滑らかにつながる
6. 「とめる」で音を止める

## 開発

```bash
npm install
npm run dev    # https://localhost:5173 (実機テスト用にLAN公開+自己署名HTTPS)
npm test       # ユニットテスト (Vitest)
npm run build  # 本番ビルド → dist/
```

iPhone実機で試すときは、同じWi-FiにつないでdevサーバーのLAN URLをSafariで開く。
DeviceOrientationの許可APIがsecure contextを要求するため、HTTPSが必須
（開発時は @vitejs/plugin-basic-ssl の自己署名証明書を使う）。

## デプロイ

`npm run build` の `dist/` を任意の静的ホスティング（Vercel、GitHub Pages 等）へ配置する。
HTTPS配信が必須。

## 既知の制約（v1の仕様）

- 鳴りっぱなし方式のため、机に置くと中央の音が鳴り続ける（「とめる」で停止）
- iOSのマナーモード（サイレントスイッチ）がオンだと音が出ない場合がある。検出・案内はしない
- キャリブレーションなし。±45度の固定マッピング
- 録音、音色選択、スケール切替、赤ちゃんモード、PWA化は含まない
